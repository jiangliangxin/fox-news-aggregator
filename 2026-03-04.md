## 经验教训

### ⚠️ Git 推送严重错误（2026-03-04）
**错误**：推送了整个 workspace（185个文件），包含敏感内容

**原因**：
1. 在 workspace 根目录执行 `git init`
2. 使用 `git add .` 添加所有文件
3. 没有检查 `.gitignore` 是否完善
4. 没有在推送前检查文件列表

**后果**：
- 暴露了 SOUL.md、MEMORY.md、USER.md、IDENTITY.md
- 暴露了 skills/ 目录
- 暴露了 memory/evolution/ 进化状态

**修复方案**：
1. 创建临时干净仓库
2. 只复制相关文件
3. 强制推送覆盖（`git push -f`）
4. 本地重置（`git reset --hard`）

**预防措施**：
1. ✅ 永远不要在 workspace 根目录创建 git 仓库
2. ✅ 创建独立的项目目录（`/root/.openclaw/projects/`）
3. ✅ 推送前必须检查文件列表：`git ls-tree -r --name-only HEAD`
4. ✅ 使用 `.gitignore` 排除敏感文件
5. ✅ 推送前审查：检查是否包含 SOUL.md、MEMORY.md、skills/等

### ⚠️ 文件架构设计错误（2026-03-04）
**错误**：workspace 根目录混合了敏感信息和项目代码

**问题**：
1. 敏感文件（SOUL.md、MEMORY.md）和项目文件混在一起
2. 没有清晰的隔离机制
3. 容易误推送敏感信息

**根本原因**：
1. 没有提前规划文件架构
2. 所有文件都放在 workspace 根目录
3. 没有区分"配置文件"和"项目代码"

**正确架构**：
```
/root/.openclaw/
├── workspace/      # 配置（敏感、个人、永远不推送）
└── projects/       # 项目（干净、独立、可推送）
```

**整改措施**：
1. ✅ 创建独立项目目录：`/root/.openclaw/projects/`
2. ✅ 克隆项目到独立位置
3. ✅ 清理 workspace 根目录（只保留配置）
4. ✅ 创建软链接保持兼容性
5. ✅ 完善隔离机制

**预防规则**：
1. ✅ workspace 只放 OpenClaw 配置文件
2. ✅ 项目代码必须放在 `projects/` 目录
3. ✅ 敏感信息永远不进入项目目录
4. ✅ 使用软链接连接两者
